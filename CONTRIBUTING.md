# Contributing

Thanks for your interest in improving this portfolio repository.

## What we accept
- New detection ideas (SPL/KQL/Sigma)
- Better tuning guidance (false positives, baselines)
- Improvements to playbooks and case documentation
- Additional MITRE mappings

## Quality guidelines
- Keep queries readable and commented (where possible).
- Document required fields (what logs/columns are needed).
- Include at least:
  - Purpose
  - Triage steps
  - False positives
  - Severity suggestion
  - MITRE mapping

## No sensitive data
- Do not include production logs, credentials, tokens, or customer data.
- Use redacted or synthetic examples only.

## Pull requests
- Create a branch
- Submit a PR with a short explanation and testing notes (even if “not tested, example only”).
